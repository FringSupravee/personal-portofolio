Project Overview

Date: June - September 2021
Tool: Figma
Position: Digital Product Designer

This project is during the summer internship as UX/UI designer with Ampd Energy. When I was started, Enernet was in the developing phases where it needed to deliver all the requested features as fast as possible. Hence, my primary responsibility is designing wireframes and providing the brief to engineers.




Breakdown all the works...

Activity Log

Function

Activity Log is created just for internal uses. The page will allow internal users to see all the history changes made by all users.
with the filter feature, users can filter from changes of account name, device information or project information



Alarm Center

Funciton

Alarm Center is the monitor page of all events that happen to Enertainers. The purpose of this page is for users to understand what is going on with every Enertainers, which becomes really useful when users have many Enertainer in possession. We highlight the total number of Enertainers, Errors, Warning, and offline devices.

Alarm Center (1st version)'s reflection

I opted to follow the first version with the existing layout on 'Fleet Monitor,' for which I got guidance from my manager to try to play around with the structure on this page. Hence, the bottom picture is the final version of the Alarm Center, which has a vertical-aligned box to indicate the critical data.
Also, I used white color for text in the top 4 boxes, which I intended to go in a more minimalistic direction. But it actually makes users uncomfortable to read. Alarm Center (1st Version)




Alert Page

Function

The purpose of the Alert Page is to send an alert notification to users' SMS, email, Whatsapp. We allow users to customize the criteria of Warning and Errors that occur to the Enertainer and the time length of any alerts. We have implemented the double verification and sent a demo alert message.
I received a comment from working on this page that my design is not entirely consistent, which will be a crucial aspect when working on detail-focused projects.


Documentation

Function

Documentation was created as a tutorial guide for users, including the physical product guide.

The first version started with a box that included the product's name, favorite button, date and time the document was updated, file size, and button for 'print' and 'view.'


Documentation (1st version)'s reflection

There're some features that I changed in the final version, which consists of

  • Unnecessary features: file size
  • Time-consuming features: 'favorite' button, 'like-dislike' feature

During the crafting process, I have learned that the customization features can take much more time in the coding process than the expected timeline. Since the engineers require to create the library to remember the choices of each user. Moreover, the Documentation page planned to have less than 10 documents. Users won't face the issue of cannot finding the file here.


Net Promoter

Funciton

Net Promoters is created for receiving feedback from customers. Although the dashboard already has the features of 'Customer Support' to receive comments about the dashboard, no remarks of customers don't mean that they will have a pleasant experience.
Hence, we created this pop-up to ask about experience in using our dashboard and our physical products. We plan to use the score as a KPI of the creative team and evaluate the priority of upcoming features in the future.



Release Note

Funciton

Release Note's objective is to inform customers about new dashboard updates and summarize highlights in each update. Because our dashboard is relatively new and serves a small group of numbers. Hence, we want to make sure that every idea and comment of our users are heard published in Release Note.
In the future, Release Note also plans to be another communication from Ampd Energy to customers, which will feature environmental news and monthly articles from us.


Release Note (1st version)'s reflection

I use color to differentiate the category of content. I quite like this version of Release Note. However, it is too fast to release this version since Release Note will initially feature new updates and fixed bugs.


Release Note (2nd version)'s reflection

Initially, I wanted to create a Release Note similar to the mailbox interface. However, I decided not to show it to my manager because everything looked too cramped and hard to read.


Release Note (2nd version)'s reflection

The interface looks pleasing, but there are the issues that the article box has 2 rows, which might result in confusing users, which articles is the newest.



Report Page

Function

Initially, our Business Development team needed to spend a lot of time generating the device performance report every month. Then they need to send each device reports to the user's email. Hence, Report Page is to enhance the experience for both users and the company sides.
Report Page can automatically generate the monthly report by itself, allowing users to view or download. Moreover, users can customize the report's period to suit their needs.

Report Page (1st version)'s reflection

The initial features consisted of...

  • Monthly Sort: Users can select the month that they want the reports to appear.
  • Viewers: Because Report Page can be accessed by only some users, I want to display who can view the report files and have the button for adding the new users that can access them quickly.
  • Report Table: Report table contains report name, description, file size, uploaded date, and 'download' button.

In the end, we decided to remove the Viewers section, a green circle row at the top. Because we will allow only high-level users access, which is the level of the users assigned since the registration process. Moreover, the point that I want to improve if I have more time is the Monthly Sort card, which I feel it's a bit unorganized.


What have I learned?
  1. learned to present the work to other department in orderly way, which inccluding to be considerate and understnad that for not everyone will understand the objective of each page
  2. Broke one step of self-doubt and gained confident in my work.
  3. My works still lack consistency in detail. Hence, I need to double-check for design detail and be consistent in a text (uppercase)
  4. Practice designing in other areas, e,g, printed materials, social media templates, apparel.
